# 《会来事儿》页面草图与逻辑标注

> 适用范围：电脑端 Web Demo  
> 对应文档：《会来事儿完整产品需求文档.md》《UI规范文档.md》  
> 草图目标：帮助理解页面结构、交互顺序、多轮对话逻辑和后续需求修改点。

## 1. 页面总流程

```text
首页
  ↓ 点击「开始游戏」
场景介绍页
  ↓ 点击「进入酒桌」
游戏主界面
  ↓ 玩家自由输入
反馈页
  ↓ 点击「继续」
游戏主界面下一轮
  ↓ 多轮循环，最多 3-6 轮
结局页
```

逻辑标注：

- `L1` 游戏不是选择题流程，所有关键回应都由玩家自由输入。
- `L2` 游戏主界面和反馈页会循环多轮，直到通关、风险失控、场面变冷、边界松动或达到轮次上限。
- `L3` 通关不是玩家说一句标准答案，而是最终让李总说出合格认可话术。
- `L4` 所有页面只面向电脑 Web 网页，不设计移动端页面。


逻辑标注：

| 标注 | 内容 | 逻辑说明 |
| --- | --- | --- |
| G1 | 顶部信息栏 | 显示产品、章节、场景、第 N 轮、当前回合目标。轮次支持 3-6 轮，不写死 1/2。 |
| G2 | 当前局势滚动字幕 | 首次进入时播放 4-6 秒，播放完收起到左下角“当前局势”。 |
| G3 | 王经理 | 直属领导 / 观察者，不主动替玩家兜底；玩家甩锅会影响领导评价。 |
| G4 | 李总 | 客户压力源 / 最终认可者。说话时必须看向林一。 |
| G5 | 李总头顶字幕 | 开场劝酒台词以头顶动态字幕或气泡播放，不做聊天列表。 |
| G6 | 酒桌场景 | 中央必须是包间酒桌，不是纯文本聊天框。 |
| G7 | 周姐 | 气氛缓冲者，必要时轻轻打圆场，但不替玩家完成表达。 |
| G8 | 阿诚 | 跟风放大器，可能起哄，也可能顺势热场。 |
| G9 | 林一 | 玩家角色。玩家点击发送后，输入内容先显示在林一头顶。 |
| G10 | 当前局势抽屉 | 默认隐藏，仅露出标签；hover/focus 后展开完整局势。 |
| G11 | 状态反馈抽屉 | 默认隐藏，展开后只显示文字状态和状态条，不显示裸数字。 |
| G12 | 小贴士抽屉 | 根据当前缺失表达给方向提示，不给标准答案，不可点击填入。 |
| G13 | 自由输入框 | 游戏核心操作区；不能出现 A/B/C/D 或预设回复。 |
| G14 | 发送回应 | 点击后进入“玩家头顶字幕 → AI 评估 → NPC 头顶字幕 → 反馈页”的流程。 |

### 4.1 游戏主界面交互顺序

```text
进入页面
→ G2 当前局势滚动字幕播放
→ G5 李总开场头顶字幕播放，李总看向林一
→ 输入框获得视觉焦点
→ 玩家输入并点击发送
→ G9 林一头顶显示玩家原话
→ AI 分析表达
→ NPC 按角色定位回应，台词显示在各自头顶
→ 当前回合目标、状态反馈、小贴士更新
→ 进入反馈页
```

## 7. 多轮对话循环逻辑草图

```text
┌──────────────┐
│ first_toast  │  李总第一次敬酒
└──────┬───────┘
       ↓
┌──────────────────┐
│ clarify_boundary │  引导说清事实和边界
└──────┬───────────┘
       ↓
┌──────────────────┐
│ offer_substitute │  引导给出替代动作，如以茶代酒
└──────┬───────────┘
       ↓
┌───────────────┐
│ raise_meaning │  引导抬高到感谢、合作、后续支持
└──────┬────────┘
       ↓
┌───────────────────┐
│ client_acceptance │  李总接受边界，停止劝酒
└──────┬────────────┘
       ↓
┌───────────────┐
│ 通关结局       │  李总输出合格认可话术
└───────────────┘

任意阶段：
风险过高 → 失控风险结局
边界清楚但气氛过冷 → 边界守住但场面变冷
面子足但边界松动 → 关系保住但边界松动
达到 6 轮仍未让李总认可 → 非通关复盘
```

逻辑标注：

- `M1` 每轮都允许玩家自由输入，不显示选项。
- `M2` 每轮 AI 需要判断玩家是否补齐：承接好意、明确边界、替代动作、抬高意义。
- `M3` 如果只缺一环，系统应继续引导下一轮，而不是直接失败。
- `M4` 通关必须满足：李总最终说出认可边界、接受以茶代酒、关系不破的话。
- `M5` 即使内部状态较好，只要李总没有输出合格认可话术，也不能进入通关结局。

## 9. 页面修改检查清单

后续修改需求时，可以优先检查以下问题：

- 是否仍然是电脑端 Web 页面。
- 主界面是否仍然只有自由输入，没有预设选项。
- 对话是否仍然显示在角色头顶，而不是聊天列表。
- 李总对林一说话时，是否看向林一。
- 角色定位是否清楚：李总是压力源和最终认可者，王经理是观察者，周姐是缓冲者，阿诚是跟风放大器。
- 状态反馈是否没有展示裸数字。
- 小贴士是否只给方向，不给完整答案。
- 多轮对话是否能继续引导，直到李总输出合格认可话术或进入非通关结局。
